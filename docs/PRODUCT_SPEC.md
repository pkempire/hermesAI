# Hermes Product Spec

## Product Definition

Hermes is a GTM operator for founder-led and lean outbound teams.

The user gives Hermes a brief in plain English:

- target market
- ideal persona
- offer
- constraints

Hermes should then:

1. source candidate accounts or people
2. enrich the strongest matches
3. explain why they fit
4. draft outreach
5. route execution through the launch channel

## Launch Scope

The launch product is intentionally narrow.

### In scope

- Google sign-in
- one public landing page
- one authenticated command center
- Websets-backed prospect discovery
- evidence-backed prospect review
- draft generation
- Gmail connection and send
- usage credits and paid upgrade path

### Out of scope for launch

- autonomous reply handling
- multi-inbox agent orchestration
- enterprise workflow builders
- heavy CRM sync surface area
- broad no-code automation features

## Primary User

The ideal launch user is:

- a founder doing outbound personally
- a small agency
- a recruiting or lead-gen operator
- a lean GTM team that wants leverage without building Clay-style tables

## User Experience

### Signed-out

The root page should feel like a product landing page, not the app shell.

The user should be able to:

- understand the product in one screen
- type a campaign brief immediately
- be prompted to sign in before execution

### Signed-in

The root page should feel like a clean operator workspace.

The user should be able to:

- start with a campaign brief
- see suggested prompt patterns
- run a search without visual clutter
- move from discovery to draft generation without leaving the flow

## Core Product Promise

Tell Hermes what you want.
Hermes should figure out the workflow.

## Launch Metrics

The launch metrics that matter are:

- sign-up conversion from landing page
- first search completion
- first draft generated
- Gmail connected
- first email sent
- upgrade conversion
